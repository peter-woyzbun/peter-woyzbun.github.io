Projects

Datm

Datm is a side project I'm currently working on. It is a graph-based, in-browser platform, written in Python, using the Django web framework, for data manipulation and visualization.

The user creates a project and adds datasets (CSV files), which are represented as nodes on a graph.
The user can apply "transformations" to datasets to create new datasets. A "manipulation set" transformation is an ordered list of manipulations (similar to R's dplyr package), where each manipulation type represents a basic verb of data manipulation (e.g. 'select', 'rename').
The user may also use SQL transformations to create new datasets.
Datasets can be easily joined, and joins are displayed as edges on the project graph.
The user can generate the Python/pandas code required to generate any dataset in their projects. As such, all transformations/manipulations are reproducible outside of Datm.
Any dataset can be easily visualized.

Query Graph

Query Graph is a Python package I've been working on as a side project. It's a graph based framework for creating templates for joining data across different types of relational databases (it currently supports PostgreSQL, MySQL, Microsoft SQL, and SQLite).

The key concept of Query Graph is the relationship between a 'parent' and 'child' query node. A query node is a single graph node with an associated 'query template'. Query templates are written in a template language I developed that is a superset of SQL. I cover the language in more detail a bit later. The figure below outlines how a parent node and child node interact to retrieve data.

  1. The parent node query is executed and data retrieved.
  2. The data output of the parent node is passed to the child node.
  3. The parent node data output is used to "fill in the blanks" of the child node query template, and the child node query is executed.
  4. The child node data is joined with the parent node's data.
